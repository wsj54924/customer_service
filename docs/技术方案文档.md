## 一、项目概述

### 1.1 赛题理解

本赛题要求设计一个具有多模态能力的电商客服智能体，能够：
- 基于20余份产品说明书回答产品使用类问题（中文+英文）
- 处理电商通用客服问题（退换货、物流、发票等）
- 理解用户上传的图片并结合文字回答
- 返回与答案相关的插图ID

### 1.2 系统目标

| 目标 | 描述 |
|------|------|
| 准确回答产品问题 | 基于手册内容，引用具体步骤和注意事项 |
| 正确关联图片 | 输出 `<PIC>` 标记和对应图片ID数组 |
| 处理客服问题 | 基于规则快速回答退换货、物流等通用问题 |
| API服务 | 提供标准RESTful接口供评测平台调用 |

- LLM调用小米MiMo云端API（免费额度充足）
- Embedding和Reranker模型本地运行（共约2GB显存）
- FAISS索引CPU运行（2732个向量，内存占用极小）

## 三、核心模块详解

### 3.1 文档解析模块 (`ManualParser`)

**功能**: 将21份产品说明书解析为带图片ID的文本块。

**处理流程**:
1. 读取手册文本文件
2. 分离文本内容和图片ID数组（文件末尾JSON数组）
3. 按 `<PIC>` 标记分割文本段
4. 按chunk_size=512字符切分，overlap=64
5. 将图片ID关联到最近的文本块

**特殊处理**:
- 中文手册: `<PIC>` 占位符 + 文件末尾图片ID数组
- 英文汇总手册: 按`#`标题分割，正则提取`Product_NN`格式图片ID

**统计**:
| 指标 | 数据 |
|------|------|
| 手册总数 | 21份 |
| 总字符数 | ~130万 |
| 图片总数 | 2608张 |
| 文本块数 | 2732个 |
| 平均chunk大小 | 476字符 |

### 3.2 向量检索模块

#### 3.2.1 Embedder

使用BAAI/bge-large-zh-v1.5将文本块编码为1024维向量，L2归一化后用于余弦相似度计算。

```python
embeddings = model.encode(texts, normalize_embeddings=True)
```

#### 3.2.2 VectorStore

FAISS IndexFlatIP索引，存储2732个向量：
- 构建: 将所有chunk的embedding添加到索引
- 查询: 返回top-10最相似的chunk
- 持久化: faiss.index + metadata.pkl

#### 3.2.3 Reranker

使用BAAI/bge-reranker-v2-m3对检索结果重新排序：
- 输入: (question, chunk_text) 对
- 输出: 相关性分数
- 取top-5作为最终上下文

### 3.3 问题分类模块 (`_classify_question`)

基于关键词匹配将问题分为4类:

| 类型 | 关键词示例 | 处理策略 |
|------|-----------|---------|
| product | 钻、表带、冰箱、键盘、manual | RAG检索 + SYSTEM_PROMPT |
| service | 退货、换货、退款、物流、保修 | SERVICE_PROMPT（跳过RAG） |
| mixed | 同时命中product和service | RAG检索 + SYSTEM_PROMPT |
| general | 未命中任何关键词 | RAG检索 + SYSTEM_PROMPT |

### 3.4 答案生成模块 (`LLMClient`)

**接口**: OpenAI兼容API（小米MiMo）

**模型**:
- `mimo-v2-pro`: 文本对话（产品+客服问题）
- `mimo-v2.5`: 多模态对话（用户上传图片时）

**Prompt设计**:

1. **SYSTEM_PROMPT**: 产品问题专用
   - 要求基于知识库回答
   - 规定`<PIC>`和图片ID输出格式
   - 要求"您好"开头，中文回答

2. **SERVICE_PROMPT**: 客服问题专用
   - 内置10条客服规则（退换货、物流、发票等）
   - 要求给出具体操作步骤和时间节点

### 3.5 图片ID处理

**关联策略**:
1. 检索阶段: 每个chunk自带的image_ids随文本一同返回
2. 上下文构建: 去重并截断至最多10张
3. LLM生成: 在prompt中告知可用的图片ID
4. 答案解析: 提取LLM输出末尾的JSON数组（如有）
5. 最终合并: 优先使用LLM指定的图片，否则使用检索得到的图片

## 五、数据处理流程

### 5.1 索引构建

```
手册文本 → ManualParser → 2732个chunks
                ↓
        BGE-large-zh-v1.5 → 2732个1024维向量
                ↓
        FAISS IndexFlatIP → faiss.index
                ↓
        chunks_metadata.json (文本+图片ID)
```

运行命令: `python -m scripts.build_index`

### 5.2 批量答题

```
question_public.csv (400题)
        ↓
    ChatEngine.answer() × 400
        ↓
    submission_fixed.csv (400条答案)
```

运行命令: `python -m scripts.batch_answer`

## 八、创新点与技术亮点

1. **问题类型自适应**: 自动区分产品/客服/混合问题，采用不同处理策略
2. **中英文双语处理**: 统一pipeline处理中英文手册和问题
3. **图片ID智能关联**: 基于RAG检索自动关联相关图片
4. **幻觉抑制**: 强制基于知识库回答，无知识库时诚实引导人工
5. **轻量级部署**: 8GB显存即可运行，LLM走云端API
